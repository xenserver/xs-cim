CIM Interface for XenServer™

Introduction

XenServer-CIM is an add-on component to XenServer that runs in Dom0 and provides a management interface for XenServer that is based on the Common Information Model (CIM) industry standard. The management model for virtualization is defined by the System Virtualization, Partitioning and Clustering (SVPC) Working Group of the Distributed Management Task Force Inc. (DMTF).

XenServer-CIM implements the following profiles (management models) as defined by the SVPC working group.

System Virtualization Profile

Virtual System Profile

Resource Allocation Profile

Allocation Capabilities Profile

Storage Resource Virtualization Profile

Ethernet Port Resource Virtualization Profile

Virtual Ethernet Switch Profile

It also implements CIM class extensions to enable various XenServer specific use cases.

CIM Interface for XenServer™ System Requirements

XenServer™ 5.6

Installation

XenServer-CIM Interface Installation

To install the XenServer CIM Interface on XenServer™, complete the following procedure:

Note:  Older versions of XenServer-CIM cannot be upgraded with this release. Please uninstall them separately, before proceeding further.

·         Install the 'XenServer Integration Suite' supplemental pack during the XenServer installation. The supplemental pack can also be installed after XenServer installation is complete. Please refer to the XenServer administration guide for more details on supplemental packs and their installation.

Components

The XenServer Integration Suite supplemental pack installs the following components:

OpenPegasus CIMOM broker: the CIMOM that accepts the CIM requests, calls the individual management providers and generates the CIM response

The XenServer-CIM providers : The CIM management providers to access the XenServer management interface

VNCControl.msi : An Active-X control that can be installed on windows clients to access the console of a XenServer host or VM running on a XenServer host.

Microsoft System Center Cross Platform providers: These CIM classes provide management access to the XenServer's Dom0 partition. The providers are distributed under the Microsoft Public License.

After the installation is complete, executing 'cimver' at a prompt will list the versions of all the components installed.

Useful Scripts

The following scripts are installed under /usr/sbin/

cimrestart, cimstart, cimstop : used to restart/start/stop the CIM services

cimiptables : used to open/close the ports used by the XenServer-CIM services

cimver : used to print the versions of all XenServer-CIM components

CIM Protocols Supported

The following CIM protocols are supported

CIM-XML

WS-Management

Security

By default, the OpenPegasus CIM daemon accepts unencrypted (HTTP over port 5988) and SSL encrypted (HTTPS over port 5989) WS-Management and CIM-XML requests.

To turn on/off either HTTP or HTTPS use ‘cimconfig –s enableHttpConnection=<true|false> –p’  or ‘cimconfig –s enableHttpsConnection=<true|false> -p’ then restart the CIM daemon using ‘cimrestart’

To edit the firewall settings, edit the /usr/sbin/cimiptables script.

By default, the XenServer-CIM services listed above uses XenServer authentication to allow CIM users to manage the XenServer or XenServer pool. Only authorized users managed by the XenServer administrator can access the CIM interfaces.

CIM Classes

Among others, the following clients can be used to manage XenServer using the XenServer-CIM classes

A WinRM client on Windows (uses the WS-Management protocol)

A standard CIM-XML client (pywbem for python users, wbemcli for a command line tool, etc)

For more information on the CIM classes, their properties, the methods they implement and their general use please refer to the class documentation>
